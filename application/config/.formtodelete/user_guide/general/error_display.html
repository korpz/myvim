Error Display

If validation fails you can access the error messages via:

$this->form->errors

This string will contain all errors wrapped by the strings you have set-up in the configuration file.

$this->form->error_array

This array will contain all errors in an associative array with the element name as key and its error message as value.

Note: All error messages will be generated with the element's label or name (if no label is available). However, you can change the error message output with the following methods: set_error() and set_error_label().

Inline Errors

Instead of fetching the errors as a string and displaying them somewhere on top of your form you can also display errors right next to the elements. In order to do this you need to set the respective attributes in the configuration file.

Error Order

Errors are being displayed in the order of the validation process order:

1. Form Element errors
2. Captcha / Recaptcha errors
3. File Upload errors
4. Model errors

Note: Models are only called if all validation before has passed.
