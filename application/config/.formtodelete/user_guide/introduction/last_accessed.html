Last Accessed Element

The form generation library remembers the last element you have added or altered. This way you don't have to specify the element name for methods that normally require the element's name.

$this->form
->text('username', 'Your Username')
->set_att('onchange', 'check_username()') // method altering the last accessed element ('username') is the same as $this->form
->text('username', 'Your Username')
->set_att('username', 'onchange', 'check_username()') // method altering a specific element ('username')

Methods that allow usage of last accessed element: add_class, add_error, alias, margin, rem_att, rem_class, set_att, set_error, set_error_label, set_rule, set_value

Proper Order

Be careful when using methods with the last accessed element notation. You always need to write the method after the element you would like to alter (or any other method you have used on the same element) before any other element is created.

Incorrect

$this->form
->text('username', 'Your Username')
->text('password', 'Your Password')
->set_att('onchange', 'check_username()') // method altering the last accessed element (in this case: 'password')

Correct

$this->form
->text('username', 'Your Username')
->add_class('user') // method altering the last accessed element ('username')
->set_att('onchange', 'check_username()') // method altering the last accessed element (still: 'username')
->text('password', 'Your Password')

Multiple methods can be called after each other on the same last accessed element.
